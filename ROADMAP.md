\# خارطة طريق Relo — التطوير بعد الهاكاثون



هدف الملف ده إنه يكون مرجع سريع لأفكار جاهزة تنفذها يوم بيوم، مرتبة من الأسهل للأصعب.



\---



\## 🟢 المرحلة 1 — تحسينات سريعة (يوم واحد لكل ميزة)



\- \[ ] \*\*Dark Mode\*\* — زرار تبديل بين الوضع الفاتح والداكن، يتحفظ في localStorage

\- \[ ] \*\*مشاركة النتيجة\*\* — زرار "شارك خطتك" يولّد صورة أو رابط قابل للمشاركة

\- \[ ] \*\*نسخ النتيجة كـ PDF\*\* — تحويل نتيجة Travel Advice لملف PDF قابل للتحميل

\- \[ ] \*\*دعم لغة إنجليزية للواجهة\*\* — زرار تبديل لغة (AR/EN) باستخدام i18n بسيط

\- \[ ] \*\*تحسين محرك البحث (SEO)\*\* — meta tags أفضل، Open Graph image

\- \[ ] \*\*صفحة "حفظ رحلاتي" بسيطة\*\* — تخزين محلي (localStorage) بدون تسجيل دخول



\## 🟡 المرحلة 2 — ميزات متوسطة (2-3 أيام لكل ميزة)



\- \[ ] \*\*حسابات مستخدمين حقيقية\*\* — تفعيل Supabase Auth (مسجل بالفعل بس مش مستخدم)، تسجيل دخول بإيميل أو Google

\- \[ ] \*\*Save \& Compare\*\* — قارن بين خطتين لدولتين مختلفتين جنب بعض

\- \[ ] \*\*خريطة تفاعلية\*\* — عرض الأماكن المهمة (سفارة، مطار، أماكن سياحية) على Google Maps Embed

\- \[ ] \*\*إشعارات ذكية\*\* — تذكير بمهام الـ Checklist القريبة من موعدها

\- \[ ] \*\*PWA (Progressive Web App)\*\* — يشتغل الموقع كتطبيق موبايل حقيقي

\- \[ ] \*\*تقييم ومراجعات المستخدمين\*\* — يقدر المستخدم يقيّم دقة المعلومات لكل دولة



\## 🔴 المرحلة 3 — ميزات كبيرة (تحتاج شراكات أو API خارجي)



\- \[ ] \*\*حجز فعلي حقيقي (Affiliate)\*\* — التسجيل في برامج Affiliate رسمية:

&#x20; - Skyscanner Partners

&#x20; - Booking.com Affiliate Partner

&#x20; - Rentalcars Affiliate

\- \[ ] \*\*أسعار حية للطيران\*\* — ربط API حقيقي (Amadeus, Skyscanner API) بدل الروابط الذكية الحالية

\- \[ ] \*\*دردشة مستمرة (Chat History)\*\* — يقدر المستخدم يكمل محادثة مع المساعد بدل سؤال واحد بس

\- \[ ] \*\*تطبيق موبايل حقيقي\*\* — React Native أو Flutter

\- \[ ] \*\*نظام اشتراكات (Freemium)\*\* — عدد استخدامات مجاني محدود، وميزات متقدمة مدفوعة



\---



\## 📝 ملاحظات تقنية مهمة



\- \*\*التقنية الحالية:\*\* Next.js 16 + TypeScript + Tailwind + Google Gemini API + Supabase (مُعد بس مش مُفعّل بالكامل)

\- \*\*قبل أي توسع كبير:\*\* فكّر في إضافة Rate Limiting على الـ API routes عشان متستهلكش رصيد Gemini بسرعة لو زاد عدد المستخدمين

\- \*\*الأمان:\*\* لو فعّلت حسابات مستخدمين، تأكد من استخدام Supabase Row Level Security (RLS) صح

\- \*\*الاختبار:\*\* لو رجعت لـ TestSprite، تذكر مشكلة الـ project-level instruction اللي وثقناها في LOOP.md (Iteration 3)
